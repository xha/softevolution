Customizing templates

As the CrudIgniter works with template files, is very easy customize the files and make them suitable for your needs.

The template files, are in the templates directory. You can customize the files as you like, because the CrudIgniter provides you an array called table, where all information concerning the selected table and fields.

The table array is used throughout the CrudIgniter, all information is stored in it, as when you choose a validation rule that will be added to a particular field, this rule is stored in the array. This enables you to know all the options chosen during the process of creating the layer, making the customization of templates really simple.

To customize a template, you don't need know anything beyond the array table and php, because you can program php normally within the template to create the file as you wish.

IMPORTANT:In some templates, such as those of the view, after being held the creating of the files they must contain php tags to be rendered by CodeIgniter. To make these php tags appear only after the files are generated, put in your template file the following tags:

<php></php>

These tags will be replaced respectively for <?php and ?>, after the creation of the files.

To learn more about the table array, you can use print_r() in php. Let's do it in the model template. Open the file templates/models/model.php and add the following line at the beginning of the file.

<?php print_r($table); ?>

Thus after generating a new model you can open the created file, for example users_model.php, and there at the beginning of the file will be the table array. This is because the CrudIgniter works with buffer control, preventing the contents of the template is shown on the console.

That way you can better understand the table array and easily customize your templates.

Templates from the View

In the case of view, you can have multiple templates beyond CRUD, you simply create a new file .php, within the directory templates/views, so when you create a new view, this template will be displayed as an option too. This allows you to create pages beyond CRUD!
