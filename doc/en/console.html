Console API

The CrudIgniter provides an API for working with the console, making simple the creation of new features. Where the developer does not need to know how to work with the console, it suffices that he knows this API.

IMPORTANT: All ConsoleIgniter class methods are static.

The main method of this class is the method writeQuestion, this method has the following signature.

mixed writeQuestion($question, array $options = null, $default = null, $canBeEmpty = false, $validate = true, $concat = false, $recursive = true)

This method is responsible for writing the question and return the response. It uses the methods write and read, but performs various validations, and is the recommended method to use when you want to ask something. The return can be string or int

The string parameter $question is the question that should be made.

The array parameter $options represents the options that can be chosen. Below is an example of how should be this array.

array('y', 'n')

You can also pass a default value in the above case, if we want in case the user does not type anything the y option is chosen, we must pass as a parameter default 0 which represents the index of the option.

The boolean parameter $canBeEmpty represents whether the response is mandatory.

The boolean parameter $validate is whether the response should be validated in that case you must pass options for that validation is done.

The boolean parameter $concat represents whether the response can contain several options such as when you are defining validation rules in the fields, where you can choose multiple rules.

The boolean parameter $recursive represents if as the response is not valid the question must still be made.

IMPORTANT: Note that when the answer is invalid, a message is displayed stating the error, these messages can be customized from the following methods.

void setEmptyMsg($msg)

Sets the message for when the value passed to the response is empty.

void setInvalidNumberMsg($msg)

Sets the message for when the passed value is an invalid number, or nonexistent in the passed choices.

void setInvalidNumbersMsg($msg)

Sets the message for when the value passed as a response is a multipart response and one or more numbers are wrong, or missing in the options.

void setInvalidFormatMsg($msg)

Sets the message for when the value passed as a response is an invalid format.

IMPORTANT: All the previous methods have their gets.

void write($msg, $newLine = true)

Write in the console, but is not expecting an answer like writeQuestion(). The $newLine parameter indicates if after writing the $msg, a new line should be displayed.

string read()

Just wait for a response from the console. Must be used in conjunction with the write() method. For example if you write a question in the console and want to get the response.

ConsoleIgniter::write('What is your name?');
$result = ConsoleIgniter::read();

In this case, the question was put through the write method and the response obtained through the read method.

void line() ----------------------------------------------------------------

This method writes a line in the console, only to separate information, such as the one above.

void bye()

Ends the execution of CrudIgniter. Can be used for example if it is necessary to read a file and this file is not found.
