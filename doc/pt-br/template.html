Customizando os templates

Como o CrudIgniter trabalha com arquivos de template, é muito fácil você customizar os arquivos e deixá-los adequados para suas necessidades.

Os arquivos de template, se encontram no diretório templates. Você pode customizar os arquivos como quiser, pois o CrudIgniter fornece para você um array chamado table, onde se encontram todas as informações referentes a tabela escolhida e os campos.

O array table, é utilizado por todo o CrudIgniter, todas as informações são guardadas nele, como quando você escolhe uma regra de validação que será adicionada para um determinado campo, esse regra é armazenada no array. Isso possibilita que você conheça todas as opções escolhidas durante o processo de criação da camada, tornando a customização de templates muito simples.

Para customizar um template, você não precisa conhecer nada além do array table e saber php, pois você pode programar php normalmente dentro do template para criar o arquivo como desejar.

IMPORTANTE: Em alguns templates, como por exemplo os da view, após ser realizada a criação dos arquivos, eles devem conter tags php para que sejam processadas pelo CodeIgniter. Para fazer com que essas tags php apareçam somente após os arquivos serem gerados, coloque no seu arquivo de template as seguintes tags:

<php></php>

Essas tags serão substituídas respectivamente para <?php e ?>, após a criação dos arquivos.

Para conhecer melhor o array table, você pode utilizar a função print_r() do php. Vamos fazer isso no template de model. Abra o arquivo templates/models/model.php e adicione a seguinte linha no começo do arquivo.

<?php print_r($table); ?>

Dessa forma após gerar uma nova model você poderá abrir o arquivo criado, como por exemplo users_model.php, e lá no começo do arquivo estará o array table. Isso acontece porque o CrudIgniter trabalha com o controle do buffer, fazendo com que não seja mostrado nada disso no console que está gerando a aplicação.

Dessa forma você pode conhecer melhor o array table e facilmente customizar o seus templates.

Templates da View

No caso da view, você pode ter diversos templates além do CRUD, basta você criar um novo arquivo .php, dentro do diretório templates/views, dessa forma quando você for gerar uma nova view, esse template será mostrado como uma opção também. O que permite que você crie páginas muito além do CRUD!
